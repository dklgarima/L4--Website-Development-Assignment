HTTP Questions and Answers

1. What is the Hypertext Transfer Protocol (HTTP)?

Ans. Hypertext Transfer Protocol (HTTP) is an application-layer protocol which follows client-server model for transmitting hypermedia documents, such as HTML.

2. Why is HTTP called a stateless protocol?

Ans. Because each request is executed independently of the requests that came before it, HTTP is known as a stateless protocol. This implies that once a transaction is completed, the connection between the browser and the server is also terminated.

3. Describe the three phases of establishing a session using client-server protocols such as HTTP.

Ans. The client-server protocal consists of three phases of establishment which are:

  1. The client establishes a TCP connection (or the appropriate connection if the transport layer is not TCP).
  2. The client sends its request, and waits for the answer.
  3. The server processes the request, sending back its answer, providing a status code and appropriate data.

5. What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?

Ans. HTTP response status codes show whether or not a particular HTTP request was completed successfully. It is of 5 types:

  1. Informational responses (100–199)
  2. Successful responses (200–299)
  3. Redirects (300–399)
  4. Client errors (400–499)
  5. Server errors (500–599)
  • 200- It indicates that the request was fulfilled.
  • 404- It indicates that the server has not found anything matching the URI given.

6. What are HTTP cookies? Give an example.

Ans. An HTTP cookie (also known as a web cookie or browser cookie) is a small piece of data sent by a server to a user's web browser. It's possible that the browser will save it and send it back with subsequent requests to the same server. For eg. keeping a user logged in.

7. Why are cookies a security issue?

Ans. Because cookies are kept locally, they can be tampered with by malevolent users. Cookies can also be used to hijack another user's session and therefore to commit fraud. They can also be used to track a user's surfing history.

8. Describe session hijacking and XSS.

Ans. A session hijacking attack occurs when an attacker takes control of a user's session. When you log into a service, such as your banking application, a session begins and ends when you log out. The attack is also known as cookie hijacking or cookie side-jacking since it relies on the attacker's knowledge of your session cookie.
    Cross-site scripting (XSS) is the most hazardous and common way of stealing an online session. Attackers can inject client-side scripts (usually JavaScript) into web pages by exploiting server or application vulnerabilities, enabling your browser to execute arbitrary code when it loads a compromised website.

9. Describe Cross-site request forgery (CSRF).

Ans. Cross-Site Request Forgery (CSRF) is a type of attack that causes an authenticated user to do undesirable actions on a web application. CSRF attack deceives the victim into submitting a malicious request. It takes on the victim's identity and privileges in order to perform an unwanted function on the victim's behalf.

10. What are Cross-site HTTP requests (CORS) and what requests use CORS?

Ans. CORS (Cross-Origin Resource Sharing) is an HTTP header-based system that allows a server to specify any additional origins (domain, scheme, or port) from which a browser should enable resource loading. The requests used by CORS are:

  1. Invocations of the XMLHttpRequest or Fetch APIs.
  2. Web Fonts (for cross-domain font usage in @font-face within CSS), so that servers can deploy TrueType fonts that can only be cross-site loaded and used by web sites that are permitted to do so.
  3. WebGL textures.
  4. Images/video frames drawn to a canvas using drawImage().
  5. CSS Shapes from images.
